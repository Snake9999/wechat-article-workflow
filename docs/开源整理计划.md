# wechat-article-workflow 开源整理计划

目标：
1. 把当前流程整理成别人能直接理解和使用的项目
2. 后续沉淀为 Hermes skill
3. 上传到 GitHub，方便团队成员拉取

当前建议的最小结构：

wechat-article-workflow/
├── README.md
├── .gitignore
├── .env.example
├── requirements.txt
├── 配置/
│   ├── workflow.yaml
│   ├── sources.yaml
│   ├── 改写提示词.yaml
│   └── 发布配置.yaml
├── 入口/
│   ├── 01_拉取文章.py
│   ├── 02_补抓清洗.py

│   ├── 04_推送草稿.py

├── 脚本/

├── 数据/

├── docs/
├── tests/
├── archive/
└── 输出/

开源前必须处理：
1. 不提交真实密钥
2. 不提交 .env
3. 不提交数据库
4. 不提交发布结果、抓取结果、改写结果等运行产物
5. 不提交 .venv 和 __pycache__

要变成 skill，需要补的内容：
1. 明确输入输出
2. 写清环境变量
3. 写清配置文件格式
4. 写清执行顺序
5. 写清已知坑（尤其是微信 IP 白名单、封面图、本地路径、md2wechat api_key）

GitHub 首次上传最小步骤：
1. gh auth login
2. gh repo create
3. git init
4. git add .
5. git commit
6. git push -u origin main
